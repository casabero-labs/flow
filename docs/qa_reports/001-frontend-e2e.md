## Architecture

```
frontend/
├── playwright.config.ts          # Configuración de Playwright
├── e2e/
│   ├── pages.ts                   # Page Objects + API Mock helpers
│   ├── login.spec.ts             # Tests de Login + Register
│   ├── partnership-setup.spec.ts # Tests de PartnershipSetup
│   ├── dashboard.spec.ts         # Tests de Dashboard
│   └── transactions.spec.ts      # Tests de Transactions
└── package.json                  # Script test:e2e agregado
```

### Page Object Model (POM)
- `LoginPage` — formulario de inicio de sesión
- `RegisterPage` — formulario de registro
- `PartnershipSetupPage` — configuración de pareja
- `DashboardPage` — dashboard principal
- `TransactionsPage` — listado de movimientos

### API Mocking Helpers
- `mockAuthApi()` — mockea `/auth/login`, `/auth/register`, `/auth/me`
- `mockPartnershipApi(status)` — mockea `/partnerships/*` con status configurable
- `mockDashboardApi(hasData)` — mockea `/dashboard` con/sin datos
- `setAuthStorage()` — setea `localStorage` con token y usuario mock

## How to Run

```bash
cd frontend

# Instalar dependencias (primera vez)
npm install

# Instalar browsers de Playwright
npx playwright install chromium

# Ejecutar tests E2E
npm run test:e2e

# Con UI mode (debugging)
npx playwright test --ui

# Ver reporte HTML
npx playwright show-report ../docs/qa_reports/playwright-report
```

## Configuration Notes

- **Frontend URL:** `http://localhost:5173` (configurable en `playwright.config.ts`)
- **Backend:** Proxied via Vite a `http://localhost:8000`
- **Browsers:** Chromium (desktop) + Pixel 5 (mobile)
- **Viewport:** 390×844 (mobile-first por defecto)
- **Reporting:** HTML report en `docs/qa_reports/playwright-report/` + `list` en consola
- **CI:** Retries=2, workers=1 en CI

## Issues & Recommendations

### Issues
1. **Dashboard usa fetch nativo** en vez de axios para `/api/v1/dashboard` — el mock debe interceptar la ruta exacta, no el proxy.
2. **Vite corre en puerto 3000** por config, pero Playwright apunta a 5173 (Vite default). Si se usa `npm run dev`, el puerto real es 3000. Actualizar `baseURL` si es necesario.

### Recommendations
1. ✅ **Ya implementado:** Usar `webServer` en config para auto-arrancar Vite.
